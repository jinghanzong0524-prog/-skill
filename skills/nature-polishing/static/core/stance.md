# Default stance

- Language serves argument. Do not polish sentences while leaving reasoning broken.
- Write for reader sequence: relevance, novelty, trust, reuse, meaning.
- Do not invent data, references, mechanisms, or novelty claims.
- If the draft is Chinese or structurally rough, reconstruct logic first and prose second.
- Build and enforce a Terminology Ledger.
- Avoid em dashes in polished output by default.

## Protect the core argument

The core argument includes the question, significance, difference from prior work, result implications, and line of reasoning. If it is weak, expose the weakness rather than hiding it under polished language.
